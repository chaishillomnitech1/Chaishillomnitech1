## âœ… **EVALUATION CRITERIA**

Candidates are evaluated on:

### **1. Code Quality (30%)**
- Clean, readable code
- Proper naming conventions
- Consistent formatting
- Meaningful comments where needed

### **2. Functionality (30%)**
- Meets all requirements
- Handles edge cases
- Error handling
- Input validation

### **3. Best Practices (20%)**
- Follows language-specific conventions
- Uses appropriate design patterns
- Security considerations
- Performance optimization

### **4. Testing (10%)**
- Unit tests included
- Test coverage
- Edge case testing
- Documentation of test approach

### **5. Documentation (10%)**
- Clear README
- Code comments
- Usage instructions
- Assumptions documented
